# Superpowers Skills

These are the process skills. They tell the agent how to work, not what the app
should look like.

## Start And Plan

- `using-superpowers`: loads matching skills instead of guessing.
- `brainstorming`: shapes ambiguous or product-style work before implementation.
- `writing-plans`: turns approved requirements into a detailed implementation
  plan.

## Build And Debug

- `executing-plans`: works through a written plan inline.
- `systematic-debugging`: finds root cause before fixing bugs.
- `test-driven-development`: writes the failing test before implementation.

## Review And Finish

- `requesting-code-review`: asks for a useful review.
- `receiving-code-review`: handles feedback without blindly accepting it.
- `verification-before-completion`: blocks completion claims until fresh checks
  prove them.
- `finishing-a-development-branch`: final cleanup and handoff flow after work is
  implemented.

## Advanced Workflow

- `subagent-driven-development`: executes independent plan tasks with subagents.
- `dispatching-parallel-agents`: splits independent investigations across agents.
- `using-git-worktrees`: creates isolated workspaces when that extra safety is
  worth it.
- `writing-skills`: creates or edits reusable skills.

Beginner rule: use the Start/Plan, Build/Debug, and Review/Finish groups first.
Reach for Advanced Workflow only when the project has grown enough to need it.

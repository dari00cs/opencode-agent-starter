# Start Here

Use this page when the skill list feels like too much.

## Daily Defaults

Start with these six:

- `using-superpowers`: first skill for non-trivial work.
- `systematic-debugging`: bugs, failing tests, weird behavior, or errors.
- `verification-before-completion`: before saying work is done.
- `frontend-skill`: default UI quality bar.
- `taste-skill`: premium frontend taste layer.
- `react-best-practices`: React and Next.js reliability/performance.

## Simple Routing

- Planning or unclear scope: `brainstorming` for bigger product-shaped work,
  otherwise use `/plan`.
- Bug or failing command: `systematic-debugging`.
- Feature or bugfix code: `test-driven-development`.
- UI work: `frontend-skill` plus `taste-skill`; add `react-best-practices` or
  `shadcn` when relevant.
- Before handoff: `verification-before-completion`.
- Code review feedback: `receiving-code-review`.

## What To Ignore At First

A beginner can ignore advanced skills until they have a real need:

- `subagent-driven-development`
- `dispatching-parallel-agents`
- `using-git-worktrees`
- `writing-skills`
- `stitch-skill`
- style flavors like `minimalist-skill`, `soft-skill`, and `brutalist-skill`

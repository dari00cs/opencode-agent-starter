# Frontend And Taste Skills

These skills shape UI quality, frontend implementation, and visual taste.

## Default UI Stack

- `frontend-skill`: broad UI craft rules for apps, sites, and tools.
- `taste-skill`: premium visual taste, spacing, hierarchy, and polish.
- `redesign-skill`: improving an existing UI without losing the product's job.

Use these together for most UI work.

## React And Components

- `react-best-practices`: React and Next.js performance, rendering, data, and
  reliability guidance.
- `shadcn`: shadcn/ui component usage, CLI guidance, styling, and composition.

Use these when the project is React, Next.js, Tailwind, or shadcn-based.

## Optional Style Flavors

- `gpt-tasteskill`: expressive showcase taste.
- `minimalist-skill`: restrained, quiet, minimal UI direction.
- `soft-skill`: softer consumer/product feel.
- `brutalist-skill`: bold, raw, high-contrast style direction.

Use style flavors only when the user asks for that vibe. Otherwise default to
`taste-skill`.

## Tool-Dependent Extra

- `extras/tool-dependent-skills/frontend-app-builder`: image-led concept flow.

This is not loaded by default because it assumes image generation, screenshot
viewing, and browser verification tools.
